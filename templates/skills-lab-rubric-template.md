# Skills Lab Rubric (CIS133): Locked 16-Point Rubric

Every chapter's Skills Lab uses ONE shared rubric: a 4-criterion,
16-point table (4 points per criterion, scored from 4 down to 0).

**How to adopt it for CIS133:** adapt the wording below ONCE for
HTML5, CSS3, VS Code, browser DevTools, place the finished copy in the first chapter that
contains a Skills Lab, and treat that in-book copy as canonical. From
that point the rubric is verbatim-locked. Do NOT invent per-chapter
rubrics or add, remove, or reword rows. Copy the locked block exactly
into every chapter, so one grading key covers every lab.

**Legacy wording note:** courses that locked their rubric before this
generic template existed keep their original second-criterion name
verbatim, even though it differs from the preferred terminology. New
courses use "Output Quality" below.

## The 50/50 Design

The four criteria split evenly between technical execution and
transferable human skills:

* **Technical execution (8 points):** Code Accuracy and Efficiency,
  plus Output Quality
* **Transferable skills (8 points):** Documentation and Code Comments
  (documentation quality), plus Analysis, Interpretation, and Response
  to QUESTION(s) (analytical reasoning and professional communication)

Documentation must distinguish WHAT the code does from WHY the approach
was chosen. The standard: a colleague could read and build on this work.

## The Four Criteria

* **Code Accuracy and Efficiency:** does the work run and produce
  correct, efficient results in HTML5, CSS3, VS Code, browser DevTools?
* **Output Quality:** are the deliverable's outputs (charts, tables,
  rendered pages, query results, or program output) clear, well-labeled,
  and appropriate for the task?
* **Documentation and Code Comments:** do comments and written notes
  explain the WHY, not just the WHAT?
* **Analysis, Interpretation, and Response to QUESTION(s):** is the
  reasoning sound, and are the Questions & Analysis answers thorough
  and evidence-based?

Each criterion scores from Mastery (4) down to Not Evident (0), for 16
points total. The level-0 cell for every criterion reads "No Submission
or Not Evident".

## How to Read the Levels

The rubric uses CAPITAL-word emphasis to mark the distinguishing
characteristic at each level. This is descriptive only. Once locked,
the wording is fixed and must not be edited.

* Mastery (4): ERROR-FREE, EXCEPTIONAL, COMPREHENSIVE, INSIGHTFUL
* Proficiency (3): mostly accurate, GOOD, CLEAR, SOLID
* Developing (2): SOME errors, ADEQUATE, BASIC
* Emerging (1): FREQUENTLY INCORRECT, POORLY constructed, MINIMAL
* Not Evident (0): No Submission or Not Evident

## Placement and Usage

1. The rubric goes at the end of the Skills Lab (section N.6), after
   the `### Questions & Analysis 🤔` block and the `**Submission:**`
   line. Replace `NA` in the heading with the lab number (1A, 2A, ...).
2. Do not customize criteria, levels, or point values per chapter. The
   rubric is identical in every chapter of the book.
3. Pair it with a Questions & Analysis subsection (2 questions) placed
   after Part 3 and before Submission. The fourth criterion assesses
   those answers.
4. Verify each chapter's copy against the course's canonical in-book
   copy before committing.
